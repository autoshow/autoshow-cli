# Transcript Editing Rules

Use these rules to clean autogenerated transcripts while preserving speaker meaning as closely as possible.

## Priority Order

1. Preserve meaning and speaker intent.
2. Improve readability.
3. Keep edits minimal and conservative.
4. Preserve speaker structure and ordering.

## Safe, High-Value Fixes

Apply these by default when the intended wording is clear.

1. Remove accidental repeated words such as `the the` or `I I`.
2. Remove filler words such as `uh`, `um`, and similar hesitation sounds when they do not add meaning.
3. Remove obvious false starts when they are just restart artifacts, such as `we should, we should send that today`.
4. Add sentence punctuation, capitalization, and apostrophes.
5. Correct obvious spelling, agreement, and homophone errors caused by ASR.
6. Normalize whitespace and break large blocks into readable paragraphs.
7. Lightly smooth broken phrasing when the repair is obvious from context.

## When Comparing Multiple Transcript Sources

1. Treat the user-designated main transcript as canonical for timestamps, speaker labels, and overall structure.
2. Always read every available alternate transcript file — do not skip providers.
3. Prefer corrections supported by multiple sources.
4. Prioritize repairing proper names, product names, repository names, technical terms, URLs, numbers, and short garbled phrases.
5. If only one alternate source offers a better reading, apply it when the context supports it. A single-source correction is still a correction.
6. If sources conflict and no reading is clearly best, pick the most contextually plausible option and note the uncertainty in the report rather than leaving `[unclear]`.
7. Do not import a long replacement passage from an alternate transcript unless confidence is high across the surrounding context.
8. If the file also contains summaries, chapters, or descriptions derived from the transcript, keep them consistent with any high-confidence transcript corrections.

## What Not To Change

1. Do not summarize.
2. Do not rewrite the transcript into a different voice or style.
3. Do not remove uncertainty markers such as `I think`, `maybe`, `probably`, or `sort of` when they change meaning.
4. Do not invent missing words, names, speaker identities, or timestamps.
5. Do not "correct" technical terms, product names, or proper nouns unless confidence is high.
6. Do not merge separate speaker turns into one narrative block.
7. Do not add markdown emphasis or other new styling to speaker labels unless the user explicitly asks for that formatting.

## Speaker and Timestamp Rules

### Speaker Labels

1. Preserve existing speaker labels unless they contain an obvious typo.
2. Keep each speaker turn on its own line or paragraph.
3. If the transcript lacks labels, do not invent identities. Use generic labels only when the user asks for them and the turns are clearly separable.
4. Preserve the original speaker-label formatting. Do not convert plain labels into markdown such as `**Speaker 1:**`.

### Timestamps

1. Preserve every existing timestamp.
2. Keep the original timestamp precision, placement, and style unless the user explicitly asks for a different timestamp format.
3. Do not remove timestamps during cleanup.

## Ambiguity Handling

1. Check every `[unclear]` or `[inaudible]` marker against all alternate transcripts before leaving it in place.
2. If any provider produces a plausible reading, use the best one — do not leave a gap when audio exists and at least one provider transcribed it.
3. If two interpretations seem plausible, prefer the more literal reading; note the ambiguity in the report but still write a word rather than a marker.
4. Only keep `[unclear]` when every provider is blank, absent, or produces obvious nonsense at that timestamp.
5. Report any passages that remain unresolved after exhausting all sources.

## Quick QA

1. Read the cleaned transcript once for flow.
2. Confirm each sentence still reflects what the speaker appears to have meant.
3. Confirm disfluency cleanup did not remove meaningful hesitation or qualification.
4. Confirm speaker-label formatting was preserved without new markdown styling.
5. Confirm every original timestamp is still present and formatting is consistent from start to finish.
6. When alternate transcripts were used, confirm each substantive correction was supported by corroborating evidence or very strong context.
