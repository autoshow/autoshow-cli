## Quick Start

```bash
# Run all tests
bun t

# Remove test-output run artifacts when successful
bun t --cleanup

# Run by tier
bun t --tier smoke
bun t --tier local
bun t --tier api
bun t --tier slow           # runs both slow-local and slow-api
bun t --tier slow-local
bun t --tier slow-api
bun t --tier smoke,local

# Price preflight suites
bun t --test-price
bun t --tier smoke --test-price
bun t --tier api --test-price --budget 25

# Run tests with a suite budget cap (cents)
bun t --budget 5
bun t --tier api --budget 25
```

## Tier System

| Tier | Description | Examples |
|------|-------------|----------|
| `smoke` | Fast/local checks with minimal dependencies | validation paths, local price checks |
| `local` | Requires local runtimes/models, no API keys | whisper local, llama local, local TTS |
| `api` | Requires API keys/network | provider service tests, integration tests |
| `slow-local` | Heavy local-only tests (no network) | reverb, large whisper, PaddleOCR image, llama Qwen3 |
| `slow-api` | Long-running network-dependent tests | setup downloads, streaming/batch downloads |

`--tier slow` is a shorthand that expands to both `slow-local` and `slow-api`.

Tier routing is defined in `test/test-runner/constants.ts`.

## Path Filtering

Run specific directories/files by passing paths directly:

```bash
bun t test/test-cases/e2e/step-2-transcribe-e2e/whisper/
bun t test/test-cases/e2e/step-3-write-e2e/openai/openai-models.test.ts
```

Path filters can be combined with runner flags.

## Run Artifacts

Each `bun t` invocation creates a timestamped run directory under `./test-output/`:

```text
./test-output/YYYY-MM-DD_HH-MM-SS_test-run/
  runner.log
  commands.log
  metrics.ndjson
  junit.xml
  report.json
  e2e-report.json
  model-calibration.json
```

- Artifacts are preserved by default.
- Use `--cleanup` to remove the run directory on successful runs.
- Runner output and streamed `bun test` lines are always prefixed with wall-clock plus elapsed time for profiling.
- Command-generated output still goes to `./output/`.
- Successful test-mode runs also emit `model-calibration.json` and may update per-model `estimation` fields inside the model config JSON files.

## Environment Variables

| Variable | Default | Effect |
|---|---|---|
| `AUTOSHOW_TEST_PRESERVE_ARTIFACTS=0` | preserve | Set to `0` to delete `./output/` directories created during the test run after each test completes |

```bash
# Keep output dirs after each test (default)
bun t test/test-cases/e2e/step-4-tts-e2e/

# Delete output dirs after each test
AUTOSHOW_TEST_PRESERVE_ARTIFACTS=0 bun t test/test-cases/e2e/step-4-tts-e2e/
```

## Current Test Layout

- `test/test-cases/e2e/` contains end-to-end command tests, including step suites, music/video/image, and integration coverage.
- `test/test-cases/validation/` currently contains model option validation coverage.
- `--price` coverage is generated by the custom test runner and helpers, not by a dedicated test-case directory.
- Local image generation is unsupported, so there is no local image test suite or local image test doc.

## Price Estimates

Price coverage is generated from the current model registry and test-runner command builders.

### Sources of Truth

- `test/test-runner/price-commands.ts`
- `test/test-utils/api-cheap-config.ts`

`api-cheap-config.ts` builds the lowest-cost representative provider/model selections dynamically from the live registry. `price-commands.ts` uses those selections as part of the broader tier-specific and scenario-specific `--price` checks.

### Run Price Preflight

```bash
# all tiers
bun t --test-price

# tier subsets
bun t --tier smoke --test-price
bun t --tier local --test-price
bun t --tier api --test-price
bun t --tier slow --test-price       # runs both slow-local and slow-api
bun t --tier slow-local --test-price
bun t --tier slow-api --test-price
```

### Run With Budget Cap

`--budget <whole-number-cents>` runs a pricing preflight for the resolved suite. In normal test mode it prints RUN/SKIP per test key (including per-variant costs when duplicate keys exist) plus a skipped-command list, and skips over-budget tests in the actual `bun test` run. Combined with `--test-price`, it marks commands under over-budget test keys as skipped in the price report without running tests.

```bash
bun t --budget 5
bun t --tier api --budget 25
bun t test/test-cases/e2e/step-3-write-e2e/openai/ --budget 5
bun t --tier api --test-price --budget 25
```

### Where Results Go

Each run writes summary artifacts to:

```text
./test-output/YYYY-MM-DD_HH-MM-SS_test-run/
  report.json
  e2e-report.json
  model-calibration.json
  commands.log
```

- `report.json` is the full runner artifact.
- `e2e-report.json` is the condensed end-to-end report with per-test service/model, run datetime, estimated/actual command duration, estimated/actual processing time, and estimated/actual cost when available.
- `model-calibration.json` records the per-model drift analysis and any config updates applied back to `src/cli/commands/models/*.json`.
