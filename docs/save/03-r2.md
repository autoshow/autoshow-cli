# Cloudflare R2 Setup and Configuration

## Overview

Cloudflare R2 is an S3-compatible object storage service with zero egress fees, making it ideal for applications with high data transfer requirements. R2 provides automatic global distribution through Cloudflare's network.

## Prerequisites

Before using the `--save r2` option, ensure you have:

1. **AWS CLI installed** (R2 is S3-compatible and uses the AWS CLI)

2. **Create R2 API Token**:
   - Go to [Cloudflare Dashboard > R2 > API Tokens](https://dash.cloudflare.com/?to=/:account/r2/api-tokens)
   - Click "Create API token"
   - Choose permissions (Admin Read & Write recommended)
   - Copy your Access Key ID (32 characters) and Secret Access Key
   - Note: R2 tokens are different from regular AWS credentials

3. **Configure AWS CLI with R2 credentials**:
   ```bash
   # Create a profile specifically for R2
   aws configure --profile r2
   ```
   When prompted, enter:
   - AWS Access Key ID: Your R2 Access Key ID (32 characters)
   - AWS Secret Access Key: Your R2 Secret Access Key
   - Default region name: auto
   - Default output format: json

4. **Set environment variables**:
   ```bash
   export AWS_PROFILE=r2
   export CLOUDFLARE_ACCOUNT_ID=your-cloudflare-account-id
   ```
   
   You can find your Cloudflare Account ID in the Cloudflare dashboard URL or on the R2 overview page.

## Configuration

### Finding Your Account ID

1. Log into the [Cloudflare Dashboard](https://dash.cloudflare.com)
2. The Account ID is visible in:
   - The URL: `https://dash.cloudflare.com/{ACCOUNT_ID}/r2`
   - The R2 Overview page sidebar
   - Account Settings page

The Account ID is a 32-character hexadecimal string like: `c6494d4164a5eb0cd3848193bd552d68`

### Creating R2 API Tokens

1. Navigate to **R2 > Manage R2 API Tokens**
2. Click **Create API token**
3. Configure token permissions:
   - **Permission**: Admin Read & Write (for full access)
   - **TTL**: Set expiration or leave unlimited
   - **IP Address Filtering**: Optional security restriction
4. Click **Create API Token**
5. **Important**: Save the Secret Access Key immediately (shown only once)

### Setting Up AWS CLI Profile

```bash
# Create a dedicated R2 profile
aws configure --profile r2

# When prompted:
AWS Access Key ID [None]: <Your R2 Access Key ID>
AWS Secret Access Key [None]: <Your R2 Secret Access Key>
Default region name [None]: auto
Default output format [None]: json
```

## Usage Examples

### Basic R2 Upload

```bash
# Set required environment variables
export AWS_PROFILE=r2
export CLOUDFLARE_ACCOUNT_ID=c6494d4164a5eb0cd3848193bd552d68

# Process and upload to R2
npm run as -- text --video "https://www.youtube.com/watch?v=MORMZXEaONk" \
  --whisper large-v3-turbo \
  --chatgpt gpt-4o-mini \
  --save r2
```

### Custom Bucket Configuration

```bash
# Use a custom bucket prefix
npm run as -- text --rss "https://ajcwebdev.substack.com/feed" \
  --last 3 \
  --save r2 \
  --s3-bucket-prefix "my-r2-content"
```

### Batch Processing

```bash
# Process multiple items and upload to R2
export AWS_PROFILE=r2
export CLOUDFLARE_ACCOUNT_ID=c6494d4164a5eb0cd3848193bd552d68

npm run as -- text --playlist "https://www.youtube.com/playlist?list=PLCVnrVv4KhXPz0SoAVu8Rc1emAdGPbSbr" \
  --deepgram nova-2 \
  --claude claude-3-5-haiku-latest \
  --save r2
```

```bash
# Test R2 access with your configured profile

  --endpoint-url https://YOUR_ACCOUNT_ID.r2.cloudflarestorage.com
```

## Integration Examples

### Access from AWS CLI

```bash
# List uploaded files
aws s3 ls s3://autoshow-c6494d4164a5eb0cd3848193bd552d68-auto/ \
  --profile r2 \
  --endpoint-url https://c6494d4164a5eb0cd3848193bd552d68.r2.cloudflarestorage.com

  --endpoint-url https://c6494d4164a5eb0cd3848193bd552d68.r2.cloudflarestorage.com

  --profile r2 \
  --endpoint-url https://c6494d4164a5eb0cd3848193bd552d68.r2.cloudflarestorage.com
```

### Generate Pre-signed URLs

```bash

  --profile r2 \
  --endpoint-url https://c6494d4164a5eb0cd3848193bd552d68.r2.cloudflarestorage.com \
```

### Using R2 with Cloudflare Workers

```javascript
// Example: Accessing R2 from Cloudflare Workers

  async fetch(request, env) {
```

## Troubleshooting

### Invalid Credential Length Error

If you get "Credential access key has length 20, should be 32":
- You're using AWS credentials instead of R2 credentials
- Create R2-specific API tokens at https://dash.cloudflare.com/?to=/:account/r2/api-tokens
- Configure a separate AWS CLI profile for R2 with the correct credentials

### Missing Account ID

If R2 upload fails with missing account ID:
```bash
# Set your Cloudflare account ID
export CLOUDFLARE_ACCOUNT_ID=c6494d4164a5eb0cd3848193bd552d68
```

### Profile Not Found

If AWS CLI can't find the R2 profile:
```bash
# Verify profile exists
aws configure list-profiles

# If missing, recreate it
aws configure --profile r2
```

### Connection Errors

If you get connection errors:
1. Verify your Account ID is correct (32 hex characters)
2. Check the endpoint URL format
3. Ensure your API token has the correct permissions
4. Try regenerating the API token

## R2-Specific Features

### Zero Egress Fees

R2's main advantage is free data transfer:
- No charges for downloading files
- No bandwidth costs for global distribution
- Ideal for content delivery and high-traffic applications

### Automatic Global Distribution

Files are automatically available through Cloudflare's global network:
- Low latency access worldwide
- Automatic caching at edge locations
- DDoS protection included

### S3 API Compatibility

R2 supports most S3 API operations:
- Compatible with existing S3 tools and SDKs
- Easy migration from S3
- Some advanced S3 features may not be supported

### Public Buckets and Custom Domains

Configure public access through the Cloudflare dashboard:
1. Go to R2 > Your Bucket > Settings
2. Enable public access
3. Optional: Configure custom domain
4. Use Cloudflare's CDN features

## Security Best Practices

1. **Use API tokens** with minimal required permissions
2. **Rotate tokens** regularly
3. **Enable IP restrictions** on API tokens when possible
4. **Use Cloudflare Access** for additional authentication
5. **Monitor usage** through Cloudflare Analytics
6. **Enable versioning** for important buckets
7. **Use Workers** for fine-grained access control

### Example: Restricting Token Permissions

When creating API tokens:
- Use **Object Read & Write** instead of Admin for basic operations
- Set expiration dates for temporary access
- Restrict to specific IP addresses or CIDR ranges
- Create separate tokens for different applications

## Cost Optimization

### R2 Pricing Structure

- **Storage**: $0.015 per GB per month
- **Class A Operations**: $4.50 per million requests (writes, lists)
- **Class B Operations**: $0.36 per million requests (reads)
- **Egress**: FREE (major advantage)

### Optimization Tips

1. **Batch operations** to reduce request counts
2. **Use lifecycle rules** to delete old files
3. **Enable compression** before upload
4. **Cache frequently accessed files** at the edge
5. **Monitor usage** in the Cloudflare dashboard

### Lifecycle Rule Example

Configure through Cloudflare Dashboard or API:
```json
{
  "rules": [{
    "id": "delete-old-files",
    "status": "Enabled",
    "condition": {
      "age": 90
    },
    "action": {
      "type": "Delete"
    }
  }]
}
```
