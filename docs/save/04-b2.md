# Backblaze B2 Setup and Configuration

## Overview

Backblaze B2 is a cost-effective S3-compatible cloud storage service offering competitive pricing and reliable performance. B2 provides strong consistency, high durability, and integration with the Backblaze CDN.

## Prerequisites

Before using the `--save b2` option, ensure you have:

1. **AWS CLI installed** (B2 is S3-compatible and uses the AWS CLI)

2. **Create B2 Application Key**:
   - Go to [Backblaze B2 Account > App Keys](https://secure.backblaze.com/app_keys.htm)
   - Click "Add a New Application Key"
   - Choose permissions (Read and Write recommended for bucket operations)
   - Copy your Key ID and Application Key
   - Note: You cannot use your master application key with the S3-compatible API

3. **Configure AWS CLI with B2 credentials**:
   ```bash
   # Configure AWS CLI with B2 credentials
   aws configure set aws_access_key_id YOUR_B2_KEY_ID
   aws configure set aws_secret_access_key YOUR_B2_APPLICATION_KEY
   ```

4. **Set environment variables**:
   ```bash
   export B2_APPLICATION_KEY_ID=your-b2-key-id
   export B2_APPLICATION_KEY=your-b2-application-key
   export B2_REGION=us-west-004  # Optional, defaults to us-west-004
   ```
   
   Common B2 regions include:
   - `us-west-004` (US West - California)
   - `us-east-005` (US East - Florida)
   - `eu-central-003` (Europe - Amsterdam)

## Configuration

### Creating Application Keys

1. Log into your [Backblaze B2 Account](https://secure.backblaze.com/user_signin.htm)
2. Navigate to **App Keys** in the left sidebar
3. Click **Add a New Application Key**
4. Configure the key:
   - **Name**: Descriptive name (e.g., "autoshow-cli")
   - **Allow access to bucket(s)**: All or specific buckets
   - **Type of Access**: Read and Write
   - **File name prefix**: Optional restriction
   - **Duration**: Set expiration or leave unlimited
5. Click **Create New Key**
6. **Important**: Save the Application Key immediately (shown only once)

### Key Capabilities Required

For AutoShow CLI, ensure your Application Key has these capabilities:
- `listBuckets` - Required for bucket operations
- `listFiles` - List files in buckets
- `readFiles` - Read file contents
- `writeFiles` - Upload files
- `deleteFiles` - Optional, for cleanup

### Setting Up Environment

```bash
# Required environment variables
export B2_APPLICATION_KEY_ID=key-id
export B2_APPLICATION_KEY=app-key

# Optional: Set specific region (defaults to us-west-004)
export B2_REGION=us-west-004

# Alternative: Use AWS credential format
export AWS_ACCESS_KEY_ID=access-key-id
export AWS_SECRET_ACCESS_KEY=secret-access-key
```

## Usage Examples

### Basic B2 Upload

```bash
# Process and upload to B2
npm run as -- text --video "https://www.youtube.com/watch?v=MORMZXEaONk" \
  --whisper large-v3-turbo \
  --chatgpt gpt-4o-mini \
  --save b2
```

### Custom Bucket and Region

```bash
# Use European region with custom bucket prefix
export B2_REGION=eu-central-003
npm run as -- text --rss "https://feeds.transistor.fm/fsjam-podcast" \
  --last 5 \
  --save b2 \
  --s3-bucket-prefix "eu-podcasts"
```

## Integration Examples

### Access from AWS CLI

```bash
# List uploaded files
aws s3 ls s3://autoshow-004b49da91f6-us-west-004/ \
  --endpoint-url https://s3.us-west-004.backblazeb2.com

# Download a specific file
aws s3 cp s3://autoshow-004b49da91f6-us-west-004/1755334409936/video.md ./local-file.md \
  --endpoint-url https://s3.us-west-004.backblazeb2.com

# Sync entire session folder
aws s3 sync s3://autoshow-004b49da91f6-us-west-004/1755334409936/ ./local-folder/ \
  --endpoint-url https://s3.us-west-004.backblazeb2.com
```

### Generate Pre-signed URLs

```bash
# Generate a pre-signed URL valid for 1 hour
aws s3 presign s3://autoshow-004b49da91f6-us-west-004/1755334409936/video.md \
  --endpoint-url https://s3.us-west-004.backblazeb2.com \
  --expires-in 3600
```

### Using B2 Native API

```javascript
// Example: Using B2 Native API with Node.js
const B2 = require('backblaze-b2');

const b2 = new B2({
  applicationKeyId: 'key-id',
  applicationKey: 'app-key'
});

async function downloadFile() {
  await b2.authorize();
  
  const response = await b2.downloadFileByName({
    bucketName: 'autoshow-004b49da91f6-us-west-004',
    fileName: '1755334409936/video.md'
  });
  
  return response.data;
}
```

## Troubleshooting

### Invalid Application Key Error

If you get authentication errors:
- Ensure you're using an Application Key, not the Master Application Key
- Application Keys must have `listBuckets` capability for bucket operations
- Create a new Application Key at https://secure.backblaze.com/app_keys.htm

### Missing B2 Credentials

If B2 upload fails with missing credentials:
```bash
# Set your B2 credentials
export B2_APPLICATION_KEY_ID=key-id
export B2_APPLICATION_KEY=app-key
```

### Region Configuration

If bucket creation fails, verify your region setting:
```bash
# Common B2 regions
export B2_REGION=us-west-004    # California
export B2_REGION=us-east-005    # Florida  
export B2_REGION=eu-central-003 # Amsterdam
```

### Permission Errors

Common permission issues and solutions:
1. **"Forbidden" error**: Key lacks required capabilities
2. **"Bucket not found"**: Key restricted to specific buckets
3. **"Invalid auth token"**: Key expired or revoked
4. **"Account cap exceeded"**: Storage limit reached

## B2-Specific Features

### Competitive Pricing

B2 offers industry-leading pricing:
- **Storage**: $6/TB/month
- **Download**: First 1GB/day free, then $0.01/GB
- **API Calls**: Class B (downloads) free, Class C (uploads) $0.004 per 10,000
- **No minimum storage duration**
- **No early deletion fees**

### Backblaze CDN Integration

Enable CDN for better performance:
1. Go to bucket settings in B2 console
2. Enable "Backblaze CDN"
3. Optional: Configure custom domain
4. Use CDN URL for public access

### Lifecycle Rules

Configure automatic file deletion:
```bash
# Via B2 Native API or Console
{
  "fileNamePrefix": "",
  "daysFromUploadingToHiding": null,
  "daysFromHidingToDeleting": 90
}
```

### Object Lock (Compliance)

B2 supports Object Lock for compliance:
- Retention periods
- Legal holds
- Compliance mode (immutable)
- Governance mode (privileged delete)

## Security Best Practices

1. **Use Application Keys** instead of Master Key
2. **Restrict key capabilities** to minimum required
3. **Set key expiration** for temporary access
4. **Restrict to specific buckets** when possible
5. **Enable 2FA** on your Backblaze account
6. **Monitor access logs** through B2 console
7. **Use server-side encryption** for sensitive data

## Cost Optimization

### Understanding B2 Pricing

Cost components:
- **Storage**: Pay for what you store
- **Egress**: 1GB/day free, then usage-based
- **Transactions**: Minimal costs for API calls
- **No charges for**: Uploads, deletes, failed requests

### Optimization Strategies

1. **Use lifecycle rules** to auto-delete old files
2. **Enable CDN** for frequently accessed content
3. **Compress files** before upload
4. **Batch API operations** to reduce transaction costs
5. **Monitor usage** in B2 console

### Cost Comparison Example

For 1TB storage with 100GB daily downloads:
- **B2**: ~$7/month ($6 storage + $1 egress)
- **S3 Standard**: ~$35/month ($23 storage + $12 egress)
- **R2**: ~$15/month ($15 storage + $0 egress)

Use different regions for redundancy:

### Bucket Replication

B2 supports cross-region replication:
1. Enable in source bucket settings
2. Select destination region
3. Configure replication rules
4. Monitor replication status

### Integration with B2 Cloud Replication

For disaster recovery:
- Automatic replication to another region
- Point-in-time recovery
- Configurable retention policies
- Cross-account replication available
